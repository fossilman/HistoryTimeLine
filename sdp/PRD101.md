# History Timezone
## 产品需求文档（PRD · MVP）
---
## 一、产品概述
### 1. 产品定位
**History Timezone** 是一个以「时间轴」为核心的历史可视化产品，帮助用户直观理解不同时代、不同文明、不同政权在时间维度上的存在与重叠关系。
本版本为 **MVP（最小可行产品）**，目标是：
* 验证时间轴模型是否直观有效
* 建立稳定的历史数据结构
* 为后续扩展（人物、AI、分析）打下基础
---
### 2. 产品目标
* 提供一个**可缩放的历史时间轴**
* 清晰展示：
  * 文明的时间范围
  * 政权的兴衰更替
  * 历史事件的发生节点
* 保证结构清晰、信息准确、可持续扩展
---
## 二、核心用户与使用场景
### 目标用户
* 历史爱好者
* 学生 / 教师
* 内容创作者（历史类视频 / 写作者）
* 对世界历史结构感兴趣的普通用户
### 核心使用场景
1. 快速了解某一文明的历史脉络
2. 对比同一时期不同政权的存在情况
3. 查阅某个历史事件在时间轴中的位置
---
## 三、核心概念定义
### 1️⃣ 文明（Civilization）
* 时间跨度长
* 作为**背景层**
* 不直接承载事件
示例：
* 华夏文明
* 欧洲文明
---
### 2️⃣ 政权（Polity）
* 文明下的具体政治实体
* 有明确起止时间
* 是事件的主要承载单位
示例：
* 汉、唐、罗马帝国
---
### 3️⃣ 事件（Event）
事件分为两种：

#### A. 持续型事件（Duration Event）

* 有起止时间
* 如战争、改革、运动

#### B. 瞬时事件（Point Event）

* 单一年份发生
* 如建国、发现、签约

---

## 四、功能需求

### 4.1 时间轴系统

#### 功能描述

* 横向时间轴
* 支持缩放（年 → 十年 → 百年）
* 支持左右拖动

#### 规则

* 时间从左到右递增
* 不同缩放级别展示不同密度的信息

---

### 4.2 文明展示（背景层）

* 以横向色带形式呈现
* 覆盖其存在的时间范围
* 位于所有内容的最底层
* 可折叠 / 高亮（非必须）

---

### 4.3 政权展示（核心层）

* 每个政权一条横向条带
* 位于所属文明之上
* 显示起止时间
* 同一文明内可并列显示多个政权

---

### 4.4 事件展示

#### 事件类型

| 类型  | 表现形式 |
| --- | ---- |
| 持续型 | 横向条  |
| 瞬时型 | 点    |

#### 交互

* 悬停：显示简要信息
* 点击：展示完整信息卡片

## 五、数据结构（MVP）

### Civilization

```json
{
  "id": "sinitic",
  "name": "华夏文明",
  "startYear": -2000,
  "endYear": null
}
```

### Polity

```json
{
  "id": "han",
  "name": "汉",
  "civilizationId": "sinitic",
  "startYear": -202,
  "endYear": 220
}
```

### Event

```json
{
  "id": "qin_unification",
  "name": "秦统一六国",
  "type": "point",
  "year": -221,
  "relatedPolities": ["qin"]
}
```
